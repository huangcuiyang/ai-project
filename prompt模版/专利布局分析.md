# Analysis Framework (核心任务)

## 1. 专利技术分类与聚类分析 (Technology Classification)
请不要只列出 IPC/CPC 代码，需结合业务场景进行**技术主题聚类**：
* **技术地形图：** 将各企业的专利划分为 3-5 个核心技术一级类目（例如：底层架构、数据处理、算法模型、应用场景等），并细分二级子领域。
* **侧重点识别：** 分析每家企业的专利主要集中在哪个技术簇？（例如：企业 A 侧重于硬件层的信号处理，而企业 B 侧重于应用层的交互逻辑）。

## 2. 专利与技术战略布局洞察 (Strategic Layout Insights)
请运用专利情报方法论，分析企业在数据背后的战略意图：
* **护城河识别 (Moats)：** 寻找**高价值专利**（特征：被引用次数高、权利要求书 (Claims) 项数多、同族专利覆盖国家多）。分析这些专利构筑了什么核心壁垒？
* **前瞻性占位 (Landmines)：** 识别企业在尚未大规模商用的新兴技术方向上的专利申请（即“占坑”行为）。
* **技术盲区与空白点 (Blank Spots)：** 对比行业通用技术栈，指出各企业在哪些关键环节缺乏专利保护，这可能构成其技术风险点。

## 3. 研发强度与演进趋势 (Temporal Trends)
* **S 曲线定位：** 基于专利申请量的时间分布，判断各企业处于技术的哪个阶段（萌芽期、爆发期、成熟期、衰退期）。
* **关键词演变：** 分析 3-5 年间，企业专利摘要中的高频关键词发生了什么变化？（例如：从“规则匹配”向“神经网络”演进）。
