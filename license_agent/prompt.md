# 一起讨论设计一款智能体

## 背景
    我开发了一款自动化设备授权系统，现在需要对接公司的各种产品。每个产品都需要进行对接、测试验证。由于产品有几十个，且每个产品有几十个版本。如果我去一一对接验证授权系统是否适配该版本的产品，将消耗我大量的工作时间。我的设备资源有限，而且我也不熟悉各个产品的安装部署和使用。因此我想设计一个智能体，然后各个产品的开发人员通过智能体完成自己产品的对接验证。

## 我期望的智能体画像
【用户】
* 产品开发人员
* 授权系统维护人员

【用户使用场景】
* 【产品开发人员】：部署好了自己的设备环境之后，告诉“智能体”我设备的相关信息（设备名称，版本和访问方式），然后让“智能体”开始进行对接测试验证
* 【产品开发人员】：“智能体”对接设备测试验证完成之后，我能获取并且下载一份测试验收报告，报告包含完整的测试验收过程和细节
* 【授权系统维护人员】：能获取到所有进行过对接测试验证的记录

## 输入
【产品形态】
* 一般是 物理设备，或者是物理设备组成的集群，都具备自己的产品门户界面

【设备授权激活过程】
1.登录设备产品界面，获取设备的硬件特征信息文件
2.登录授权系统界面，把设备特征信息文件上传给“授权系统”，然后下载“授权文件”
3.登录设备产品界面，把“授权文件”导入到设备中，设备完成授权激活

【授权系统】
* 已经把“设备授权激活过程”自动化了，用户只需要在“授权系统”界面上配置好待授权的设备信息，然后点击即可自动化完成
* 授权系统能和所有产品的设备网络互通

【产品开发人员】
* 非常熟悉自己的产品，且有丰富的设备资源

## 任务
* 仔细阅读和分析，理解“智能体”的核心目的，补充和发散，然后和我一起逐步的设计它，最终产出一份“智能体”产品项目设计书

## 要求
* 请站着“产品开发人员”的视角上，思考和发散你的使用场景
